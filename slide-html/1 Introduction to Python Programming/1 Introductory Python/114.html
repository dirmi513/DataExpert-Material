Python Syntax

Any natural language, such as English, has certain grammar rules that you must follow in order to construct valid sentences. This is also the case for all programming languages. Each unique programming language is analogous to a human language, and requires adherence to specific grammar, also known as syntax, rules. These syntax rules determine whether or not your code is properly written, and if it is not, you will most definitely get a SyntaxError when trying to execute the code. Some of Python's syntax rules that you have already come across (explicitly and implicitly) are:

  • Every distinct command must be separated by a new line in your source code.
    • In the previous slide, we showed you the output (SyntaxError) of trying to execute code that had two consecutive print() functions separated by a space, instead of a new line.
  • To properly utilize a function within your source code, it must be written in the following format: name_of_function(function arguments).
    • Lets use print(5 + 5) as an example. The function's name is print, followed by an open parenthesis (, the arguments passed to the function (which we will cover in a later lesson) 5 + 5, and finally a closing parenthesis ).
